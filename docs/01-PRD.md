# 产品需求文档（PRD）

## 项目名称

Site Manager — 网站地址管理面板

## 项目背景

在日常工作中，团队成员需要频繁访问各类网站（开发工具、文档中心、监控面板、内部系统等），目前缺乏统一的管理入口，存在以下痛点：

- 书签管理分散，无法按业务类型快速筛选
- 无法直观了解站点可用状态
- 新成员上手成本高，不知道有哪些可用工具
- 站点信息变更后无法及时同步通知

## 目标用户

- 开发团队（主要）
- 运维团队
- 产品/设计团队
- 任何需要管理大量网站书签的个人或团队

## 核心功能

### P0 — 必须实现

| 功能 | 描述 |
|------|------|
| 站点卡片展示 | 以卡片形式展示所有站点，显示名称、地址、类型、描述、状态指示灯 |
| 类型分类 | 支持按类型分组展示（开发工具、文档、API、监控、社交媒体、内部系统等） |
| 类型筛选 | 顶部筛选栏，点击类型快速过滤卡片 |
| 新增站点 | 表单录入站点信息（名称、URL、类型、描述、标签） |
| 编辑站点 | 修改已有站点信息 |
| 删除站点 | 删除站点（二次确认） |
| 搜索 | 按名称/URL/标签模糊搜索 |
| 暗色模式 | 支持亮色/暗色主题切换 |

### P1 — 重要但可延后

| 功能 | 描述 |
|------|------|
| 站点健康检测 | 定时检测站点可达性，卡片上显示在线/离线/未知状态 |
| 拖拽排序 | 卡片可拖拽调整顺序 |
| 导入导出 | 支持 JSON/CSV 格式批量导入导出站点数据 |
| Favicon 自动获取 | 新增站点时自动抓取网站图标 |
| 标签系统 | 除类型外，支持自定义标签进行多维度分类 |

### P2 — 锦上添花

| 功能 | 描述 |
|------|------|
| 网站预览缩略图 | 卡片内嵌网站截图预览 |
| 多用户支持 | 简单的账号体系，每人独立管理自己的站点 |
| 分享功能 | 生成分享链接，将站点集合分享给他人 |
| 浏览次数统计 | 记录站点点击次数，按热度排序 |
| 快捷键操作 | 键盘快捷键快速搜索、切换分类 |

## 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 首屏加载 < 2s，支持 500+ 站点流畅展示 |
| 兼容性 | Chrome、Firefox、Safari、Edge 最新两个大版本 |
| 响应式 | 支持桌面端（1280px+）和平板端（768px+） |
| 可访问性 | 基本无障碍支持（键盘导航、语义化标签） |
| 数据安全 | 本地部署，数据不出服务器 |

## 用户流程

```
打开面板 → 查看站点卡片 → 点击卡片跳转目标站点
         → 筛选类型 → 查看特定分类
         → 搜索关键词 → 快速定位
         → 新增/编辑/删除站点 → 管理站点信息
```

## 竞品参考

- Heimdall（开源仪表盘）
- Homarr（现代导航页）
- Dashy（自托管仪表盘）

本项目的差异化：更轻量、更专注卡片管理体验、中文优先、开箱即用。
